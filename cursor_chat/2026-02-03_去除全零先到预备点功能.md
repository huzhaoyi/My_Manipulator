# 2026-02-03 去除「角度全零时先到达一个点」功能

## 需求
用户确认：是否有「角度全 0 时要先到达一个点」的逻辑；若有则去除，因为已有「先到预抓取」步骤。

## 结论
- 当前代码里**没有**实际执行「全零时先到预备点」的逻辑；只有历史遗留的**参数**（`auto_move_to_ready`、`ready_joint1~4`、`near_zero_threshold`）在配置和参数管理里声明/加载/打印，未被任何执行路径使用。
- `task_context.cpp` 里的「全零」仅用于**预抓取阶段**：检测 `is_near_zero_state` 后决定用关节目标还是位姿目标（避免 OMPL 从奇异点采样失败），**不会**多一步「先到预备点再预抓取」。

## 修改内容（移除未用参数）

1. **cable_grasp.yaml**  
   删除「全零状态预运动」整块：`auto_move_to_ready`、`ready_joint1~4`、`near_zero_threshold`。保留 `pregrasp_joint*_deg`（网页「到达预抓取位置」按钮用）。

2. **parameter_manager.hpp / parameter_manager.cpp**  
   删除上述 6 个参数的声明、加载及日志输出（「全零状态预运动: 启用/禁用, 预备位置: [...]」）。

3. **grasp_types.hpp**  
   在 `GraspParameters` 中删除 `auto_move_to_ready`、`ready_joint1~4`、`near_zero_threshold`。

## 验证
- `colcon build --packages-select m5_grasp` 通过。

流程仍为：打开夹爪 → **直接执行预抓取（move to pregrasp）** → 后续 align/descend/…，无「全零先到预备点」步骤。
