# 2026-02-03 去除地面障碍与视觉负 z

## 需求
- 去除地面障碍，现有支架做测试。
- 视觉可以发负数（如 position.z 为负，表示基座/支架下方目标）。

## 修改内容

### 1. 默认不添加地面碰撞体
- **parameter_manager.cpp**：`declare_parameter("grasp.add_ground_plane", false)`，默认改为 `false`。
- **grasp_types.hpp**：`add_ground_plane{false}`，默认改为不添加地面。

### 2. 启动时移除已有地面障碍
- **m5_grasp.cpp**：在 `loadParameters()` 之后，若 `add_ground_plane == false`，对 `planning_scene_` 发送 `CollisionObject::REMOVE`，移除 id 为 `"ground_plane"` 的碰撞体，并打日志「已移除地面碰撞体 ground_plane（支架测试模式）」。
- 增加 `#include <moveit_msgs/msg/collision_object.hpp>`。

### 3. 视觉位姿不限制 z（允许负数）
- 代码中缆绳位姿来自 `msg.position`，未做 z 的 clamp 或过滤，**视觉已可发负 z**。
- **task_context.cpp**：在构造缆绳位姿处增加注释，说明 `position.x/y/z` 直接使用视觉值，z 可为负（支架/基座下方）。
- **cable_grasp.yaml**：在 `add_ground_plane` 与地面相关注释中注明「false=去除地面障碍，支架测试、视觉 position.z 可为负」。

## 验证
- `colcon build --packages-select m5_grasp` 通过。
